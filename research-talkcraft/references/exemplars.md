# Exemplars

This file is the index for the exemplar collection. Exemplars provide concrete anchors for what weak and improved scripts look like across domains, audiences, and failure modes.

## Index

### By Domain

- `exemplars/cs-ml.md` — Computer science and machine learning examples (motivation, methods, results, conclusion).
- `exemplars/finance-econ.md` — Finance and economics examples (identification, magnitude, causal language).
- `exemplars/neuroscience.md` — Neuroscience and biology examples (methods intuition, mechanism explanation).
- `exemplars/social-science.md` — Social science and mixed-methods examples (qualitative framing, policy implications).

### By Audience and Occasion

- `exemplars/seminar-style.md` — Seminar presentation examples (mixed-audience hook, intuition first).
- `exemplars/defense-style.md` — Thesis defense examples (contribution boundary, methodological justification).
- `exemplars/group-meeting.md` — Informal group meeting examples (honest status update, clear ask).
- `exemplars/mixed-background.md` — Mixed-background stakeholder examples (dual-level explanation, jargon bridging).

### By Failure Mode

- `exemplars/failure-modes.md` — Common failure patterns and their fixes (bullet reading, overclaiming, missing intuition, timing overrun, contradiction mishandling, weak contribution framing).

## How to Use These Exemplars

1. **During critique:** Ask whether the script sounds more like the weak or improved version in the relevant exemplar.
2. **During revision:** Match the weakness to the relevant exemplar category and apply the same transformation.
3. **During Q&A generation:** Use the exemplar scenario as a source of likely questions.
4. **During rehearsal checklist creation:** Check whether the user can deliver the improved version naturally.

### Cross-Domain Adaptation

These exemplars span multiple fields, but the revision principles are domain-agnostic. When working with decks from any discipline, apply the same transformations:
- Replace bullet reading with explanation and interpretation.
- Replace method detail dumps with audience-oriented intuition.
- Replace result statements with comparative interpretation.
- Replace vague contribution claims with evidence-backed framing.
- Add narrative transitions between slides.
- Define jargon inline for mixed audiences.
